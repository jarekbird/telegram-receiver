# PHASE2-087: Register async handler

**Section**: 10. TelegramMessageJob Conversion
**Subsection**: 10.11
**Task ID**: PHASE2-087

## Description

Set up async handler registration for TelegramMessageHandler to handle message processing in TypeScript/Node.js. Since this application doesn't use a queue system, we leverage Node.js's asynchronous nature to process messages directly. This task converts the Sidekiq job queue infrastructure from Rails to async handler patterns in Node.js.

The Rails implementation uses:

- **Sidekiq** as the ActiveJob adapter (configured in `jarek-va/config/application.rb` line 31)
- **Sidekiq initializer** (`jarek-va/config/initializers/sidekiq.rb`) for Redis connection and configuration
- **Sidekiq YAML config** (`jarek-va/config/sidekiq.yml`) for queue and concurrency settings
- **Sidekiq worker process** started separately (`bundle exec sidekiq` or Docker service)
- **TelegramMessageJob** uses `queue_as :default` (line 11 in `jarek-va/app/jobs/telegram_message_job.rb`)

The TypeScript/Node.js implementation should use:

- **Async handler pattern** (using Node.js native async/await)
- **Async utilities** from `src/utils/async.ts` for retry and error handling
- **Base async handler** from `src/handlers/base-async-handler.ts` (if using base class pattern)
- **Direct async execution** instead of queue-based processing
